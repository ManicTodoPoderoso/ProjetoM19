Desenvolvimento Ágil de Software

a)Conceitos Básicos

Desenvolvimento Ágil de Software é um termo que define um conjunto de metodologias (Conjunto Estruturado de Práticas) utilizadas no desenvolvimento de software. O termo “ágil” tornou-se popular pelo Manifesto para o Desenvolvimento Ágil de Software, cujos valores e princípios sustentam uma vasta série de estruturas de desenvolvimento de software, incluindo o SCRUM e KANBAN.

b)Princípios

Os principais valores do Desenvolvimento Ágil de Software são: • Garantir a satisfação do consumidor entregando rapidamente e continuamente softwares funcionais • Cooperação diária entre desenvolvedores • Garantir a simplicidade e organização • Reuniões regulares com a equipa para se tornarem mais eficazes

Metodologia Ágil – SCRUM

a)Visão Geral

SCRUM é a estrutura mais utilizada para o desenvolvimento ágil de software. As funcionalidades a serem implementadas num projeto são mantidas em uma lista chamada Product Backlog. Os criadores do SCRUM, Ken Shwaber e Jeff Sutherland escreveram o The Scrum Guide para que as pessoas aprendam a utilizá-lo de forma clara. Neste guia, o SCRUM é definido como sendo leve, simples de entender mas ao mesmo tempo difícil de dominar.

b)Funcionamento dos Ciclos e Fases de Desenvolvimento

No SCRUM os projetos são divididos em ciclos que se chamam Sprints. No início de cada Sprint faz-se uma reunião de planeamento (Sprint Planning Meeting) onde o Product Owner (quem define os itens que compõem o Product Backlog) escolhe quais são os itens prioritários do Product Backlog e a equipa seleciona as atividades que irão realizar durante o primeiro Sprint. No início de cada dia é feita uma reunião (Daily Scrum Meeting) onde a equipa fala sobre que progressos foram feitos no dia anterior, o que irão fazer no dia atual e que desafios enfrentaram / vão enfrentar. Quando um Sprint for terminado, a equipa apresenta as funcionalidades implementadas numa última reunião (Sprint Review Meeting) e o ciclo reinicia-se.
